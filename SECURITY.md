# Security Policy for ShipKit

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

ShipKit's CLI has **zero runtime dependencies** — it uses only Node.js built-in modules (fs, path, readline, https). This dramatically reduces the supply-chain attack surface.

However, if you discover a security vulnerability:

1. **Do not open a public issue.**
2. Email: [hello@sagargiri.com](mailto:hello@sagargiri.com)
3. You should receive a response within 48 hours.
4. If the issue is confirmed, a fix will be released within 7 days.

## Scope

The following are covered by this policy:

- The `shipkit-pipe` npm package
- The template files generated by the CLI
- GitHub Actions workflows generated by the CLI

The following are **out of scope**:

- Vulnerabilities in tools that ShipKit configures (GitHub Actions, Husky, etc.)
- Vulnerabilities in your own project code

## Supply Chain

ShipKit CLI has **zero npm dependencies** (only Node.js built-ins). The generated config files reference standard tools (GitHub Actions, Husky, lint-staged, CodeQL). Keep those tools updated via Dependabot (configured in `.github/dependabot.yml`).
