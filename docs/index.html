What it actually does

Not another boilerplate. An automated team that runs after you push.

⚡

Smart CI/CD

Reads your package.json scripts. Only generates CI steps for what exists. No lint script? No lint step. Your CI never fails because of missing commands.

🏥

Health Monitoring

Pings your site every 6 hours. If it's down, auto-creates a GitHub Issue. When it recovers, auto-closes the issue. You know before your users do.

🔒

Security Scanning

CodeQL scans for vulnerabilities on every push. Dependabot keeps your dependencies updated. No manual checking needed.

🤖

AI Agent Config

Generates AGENTS.md so your AI coding agent (Claude, Cursor, Copilot, any) knows your stack, your rules, and your project state.

✅

Verify Command

npx shipkit-pipe check — pings your deploy URL, validates CI config, checks for vulnerabilities. One command tells you if everything works.

📦

Works Everywhere

Monorepos, empty directories, Next.js, Express, Vue, Svelte, Go, Python — detects what you have and generates accordingly. Zero config.

How it works

Three seconds from install to working pipeline.

1

Run the command

npx shipkit-pipe in any project directory. No install, no config file, no setup wizard.

2

Auto-detect

Reads your package.json, lock files, git remote, deploy config. Figures out your stack, package manager, Node version, and available scripts.

3

Generate pipeline

Creates GitHub Actions workflows, health checks, security scanning, and AI agent config — only what your project actually needs.

4

Push and forget

git push — CI runs on every PR, health checks run every 6 hours, Dependabot keeps deps fresh, CodeQL scans for vulnerabilities.
